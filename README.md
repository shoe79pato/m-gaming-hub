# M Gaming Hub

一个用于归档和发布独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放多个独立的 HTML 页面，这些页面以静态形式存在，不依赖特定后端服务。每个页面通常代表一个独立的功能模块、工具或展示内容。

## 目录结构

```
.
├── pages/          # 存放所有 HTML 页面文件
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/         # 公共资源文件（CSS、JS、图片等）
└── README.md       # 本文件
```

- `pages/`：主要目录，所有独立 HTML 页面均放置于此。
- `assets/`：页面可能引用的公共资源，如图片、样式表和脚本。

## 页面归档说明

- 每个 HTML 页面均为自包含文件，可直接在浏览器中打开运行。
- 页面之间无耦合关系，可独立维护和更新。
- 部分页面可能引用 `assets/` 下的公共资源，请确保文件路径正确。
- 页面名称应尽量反映其功能或内容，方便识别。

## 维护说明

- 添加新页面：在 `pages/` 下创建新的 HTML 文件，如有需要可将相关资源放入 `assets/`。
- 修改已有页面：直接编辑对应 HTML 文件，注意保持页面自包含性。
- 删除页面：移除 `pages/` 下对应文件，同时清理不再使用的资源文件。
- 不建议在页面中嵌入外部依赖或动态数据接口，以保持静态归档的稳定性。

## 贡献

欢迎提交 Pull Request 或 Issue 来改进页面内容或结构。请确保新增页面不包含任何敏感或违规信息。
